Available Shortcodes

WpBusTicketly provides several shortcodes to display bus search forms, bus listings, and ticket views on any page or post. Each shortcode accepts optional parameters for customisation.

Bus Search Form

[wbtm-bus-search-form]

  • Renders the main bus search interface with origin, destination, and date fields
  • Place this on a page where customers begin their search
  • Recommended page slug: /bus-search-list/
  • Automatically redirects results to the page containing [wbtm-bus-search]
  • Styling can be customised via the Bus Manager colour settings

Bus Lists / Search Results

[wbtm-bus-search]

  • Displays bus search results based on the user's origin, destination, and date
  • Must be placed on the same page referenced by the search form action
  • Shows available buses, departure times, seat availability, and pricing
  • Each bus entry links to the seat selection and booking flow
  • Supports pagination when many buses are available for a route

Additional Shortcodes

Beyond the core search pair, the plugin offers shortcodes for listing buses and displaying purchased tickets.

Bus Lists

[wbtm-bus-list]

  • Displays a static list of all available buses without requiring a search
  • Useful for route overview pages or sitemap-style navigation
  • Supports parameters: style, order, orderby, posts_per_page
  • Example: [wbtm-bus-list style="grid" order="asc" posts_per_page="10"]
  • Buses link to their individual booking pages

View Ticket

[wbtm-view-ticket]

  • Enables customers to look up their booking details using an order ID and email
  • Renders a simple form that fetches ticket information
  • Displays passenger details, seat numbers, departure time, and route
  • Ideal for a dedicated "View Ticket" page accessible from the menu
  • Works with the PDF ticket addon to display download links

Shortcode Parameters

Fine-tune the behaviour of each shortcode with the following optional attributes.

Common Parameters

  • style — Display style: list (default) or grid
  • order — Sort order: asc or desc
  • orderby — Sort field: date, title, price
  • posts_per_page — Number of buses per page (default: -1 for all)
  • bus_type — Filter by bus type slug (e.g. ac, non-ac)

Examples

  • [wbtm-bus-list style="grid" orderby="title" posts_per_page="6"]
  • [wbtm-bus-search-form] (used with default settings)
  • [wbtm-view-ticket] (no parameters required)
  • Combine multiple shortcodes on the same page for complex layouts

Page Setup Best Practices

Follow these guidelines to ensure your bus search system works correctly.

Page Structure

  • Create a page titled "Bus Search" with slug /bus-search-list/
  • Place [wbtm-bus-search-form] on the search page
  • Place [wbtm-bus-search] on the same page or a dedicated results page
  • Create a "View Ticket" page with slug /view-ticket/ and the [wbtm-view-ticket] shortcode
  • Add these pages to your navigation menu for easy access

Troubleshooting Shortcodes

  • If search results don't appear, re-save permalinks at Settings → Permalinks
  • Ensure no caching plugin is serving stale shortcode output
  • Verify the shortcode is not wrapped in paragraph tags by the editor
  • Use a classic WordPress editor or a shortcode block for reliable output
  • Check that the bus search form action URL points to the correct results page

Page Builder Integration

WpBusTicketly shortcodes work with most popular page builders. Use the native shortcode widget or element to embed them.

Elementor

  • Drag the Shortcode widget onto the page
  • Paste the shortcode (e.g. [wbtm-bus-search-form]) into the widget field
  • Save and preview the page to verify rendering
  • Style the surrounding section with Elementor's design controls

Gutenberg Block Editor

  • Add a Shortcode block from the block inserter
  • Enter the shortcode directly into the block content area
  • Use the Code block if the shortcode is being wrapped in paragraphs
  • Preview the page to confirm the shortcode output renders correctly
