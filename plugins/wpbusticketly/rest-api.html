API Overview

WpBusTicketly exposes REST API endpoints that allow external applications to retrieve bus schedules and passenger data. All requests must be authenticated using standard WordPress credentials.

Authentication

  • Use Basic Auth with WordPress username and application password
  • Generate an Application Password at Users → Profile → Application Passwords
  • Include the credentials in the Authorization header:
  • Authorization: Basic base64(username:application_password)
  • OAuth 1.0a is also supported via WordPress REST API infrastructure

Base URL

  • All API endpoints are prefixed with the WordPress REST API namespace:
  • https://your-site.com/wp-json/wbtm/v1/
  • Replace your-site.com with your actual domain name
  • Test connectivity by accessing the root endpoint
  • Responses are returned in JSON format

Available Endpoints

The following REST API endpoints are available for retrieving bus and booking data.

Get Bus List

GET /wp-json/wbtm/v1/buses

  • Returns a list of all published buses with their details
  • Each bus entry includes: ID, title, type, route, departure time, fare
  • Optional query parameters: ?route_id=5, ?type=ac
  • Supports pagination: ?page=1&per_page=20
  • Example response:
{
  "buses": [
    {
      "id": 42,
      "title": "Greenline Express",
      "type": "AC",
      "route": "Dhaka → Chittagong",
      "departure": "08:00",
      "fare": 550
    }
  ],
  "total": 1,
  "page": 1
}

Get Passenger List

GET /wp-json/wbtm/v1/passengers

  • Returns passenger details for all confirmed bookings
  • Filter by bus: ?bus_id=42
  • Filter by travel date: ?date=2025-12-25
  • Each entry includes: passenger name, phone, email, seat numbers, fare
  • Example response:
{
  "passengers": [
    {
      "booking_id": 103,
      "name": "John Doe",
      "phone": "+8801712345678",
      "email": "john@example.com",
      "seats": ["A1", "A2"],
      "bus": "Greenline Express",
      "travel_date": "2025-12-25",
      "total_fare": 1100
    }
  ],
  "total": 1
}

Additional Endpoints

Extended endpoints available for more granular data access.

Single Bus Details

GET /wp-json/wbtm/v1/buses/{id}

  • Returns full details for a specific bus by ID
  • Includes seat layout, boarding points, and pricing tiers
  • Example: GET /wp-json/wbtm/v1/buses/42
  • Returns 404 if the bus ID does not exist

Route List

GET /wp-json/wbtm/v1/routes

  • Returns all available routes with start and end points
  • Useful for populating dropdown selectors in external apps
  • Each entry includes: ID, name, start stop, end stop, distance
  • Supports the same pagination and filtering parameters

Integration Use Cases

The REST API enables a wide range of integrations beyond the core WordPress front-end.

External Display Systems

  • Sync bus schedules with airport-style departure boards
  • Push real-time seat availability to digital signage
  • Display route maps on information kiosks
  • Update departure times automatically from the WordPress backend

Custom Applications

  • Build mobile apps (iOS / Android) using the REST API as a backend
  • Create custom booking widgets for third-party websites
  • Integrate with CRM systems for automated passenger data sync
  • Build reporting dashboards with live booking and revenue data

Rate Limits & Best Practices

Follow these guidelines to ensure reliable and efficient API usage.

Rate Limiting

  • WordPress REST API does not enforce strict rate limits by default
  • High-traffic applications should implement client-side caching
  • Use the If-None-Match header with ETags when possible
  • Consider using a caching plugin for frequently accessed endpoints
  • Respect server resources by limiting concurrent requests

Security

  • Always use HTTPS in production to encrypt API traffic
  • Generate unique Application Passwords per integration
  • Never expose credentials in client-side JavaScript
  • Restrict API access by IP if the integration is internal
  • Regularly audit and revoke unused application passwords
