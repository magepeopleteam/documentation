General Guidelines

This section covers customisation, internationalisation, email notifications, booking management, FAQ configuration, and best practices for running your car rental operation efficiently.

Plugin Customisation

Tailor the plugin to match your brand and workflow through the Settings panel and built-in customisation options.

Branding & Labels

  • Change the plugin public label from "Car Rental" to any custom name (e.g., "Rent-a-Car").
  • Set a custom URL slug for all rental pages (e.g., /vehicles/ instead of /car-rental/).
  • Upload a custom logo or brand icon for the booking form header.
  • Configure your business address, phone number, and email displayed on invoices.
  • Customize the booking form colours via the Style tab in settings.

Behaviour Settings

  • Enable or disable guest checkout for faster bookings without account creation.
  • Require customer account creation for all bookings.
  • Set the default booking status (Pending, Confirmed, or On Hold).
  • Choose whether to send admin notification emails on new bookings.
  • Configure automatic booking confirmation after successful payment.

Translation & Localisation

Translating the Plugin

  • Use Loco Translate to translate all plugin strings directly from the WordPress admin.
  • Language files are located in /wp-content/plugins/car-rental-manager/languages/.
  • The plugin supports both .po and .mo file formats.
  • WPML and Polylang are supported for multi-language sites.
  • Translate vehicle names, descriptions, and categories separately per language.

Regional Settings

  • Set the default currency and currency position (before/after amount, with/without space).
  • Configure date format for the booking calendar (d/m/Y, m/d/Y, etc.).
  • Choose the first day of the week for calendar displays.
  • Set the timezone to match your business location.
  • Configure measurement units (km or miles) for vehicle distance tracking.

Email Notifications

Automated email notifications keep customers and admins informed at every stage of the booking lifecycle.

Notification Triggers

  • New Booking: Sent to admin when a booking is placed.
  • Booking Confirmed: Sent to customer when booking status changes to confirmed.
  • Booking Cancelled: Sent when a booking is cancelled by customer or admin.
  • Upcoming Reminder: Sent to customer 24 hours before pickup.
  • Payment Receipt: Sent after successful payment with invoice details.

Email Customisation

  • Edit email subject lines and body content for each notification type.
  • Use shortcode placeholders: {customer_name}, {vehicle_name}, {pickup_date}, {total_price}.
  • Choose HTML or plain text email format.
  • Send test emails from the settings panel to verify delivery.
  • Integrate with SMTP plugins for reliable email delivery.

Booking Management

Manage all rental bookings from a centralised dashboard with filtering, status updates, and detailed views.

Dashboard Overview

  • View total bookings, revenue, active rentals, and fleet utilisation stats.
  • Filter bookings by status (Pending, Confirmed, Completed, Cancelled, Refunded).
  • Search bookings by customer name, vehicle, booking ID, or date range.
  • Export bookings to CSV for accounting or offline records.
  • Quick actions: confirm, complete, cancel, or refund from the list view.

Detailed Booking View

  • View full booking details: customer info, vehicle, dates, extras, and total.
  • Edit booking dates, vehicle assignment, or add-ons after placement.
  • Add internal notes visible only to staff.
  • Send a manual email to the customer from the booking edit screen.
  • View payment history and refund status for each booking.

FAQ Management

Provide a self-service FAQ section to reduce support queries.

  • Go to Car Rental > FAQ > Add New to create a new FAQ entry.
  • Group FAQs by category for better organisation.
  • Display FAQs on the front end using the [mpcrbm_faq] shortcode.
  • FAQs support WYSIWYG content for detailed answers with images or links.
  • Reorder FAQs by drag-and-drop in the admin list.

Best Practices

  • Regularly update the plugin and add-ons to the latest version for security and features.
  • Back up your database before major configuration changes.
  • Test booking flows on a staging site before deploying to production.
  • Use a caching plugin with page exclusion for booking form pages.
  • Monitor booking logs for errors or failed payment attempts.
  • Provide clear terms and conditions linked from the booking form.
