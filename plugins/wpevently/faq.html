General Questions

Common questions about installing, configuring, and using the WpEvently plugin for the first time.

How do I install WpEvently?

  • Install via WordPress admin: Plugins → Add New, search for "WpEvently"
  • Install via zip: Download from Magepeople, upload under Plugins → Add New → Upload
  • Install via FTP: Extract zip, upload folder to /wp-content/plugins/, activate
  • The Pro version requires manual zip upload — it is not available in the WP repo

Can I sell paid tickets through the plugin?

  • Yes — WpEvently integrates with WooCommerce for paid ticket sales
  • Create free or paid ticket types with different prices
  • Supports multiple payment gateways via WooCommerce
  • Set up variable pricing, early bird discounts, and group rates
  • Tax configurations are inherited from WooCommerce settings

Does the plugin support recurring events?

  • Yes — create daily, weekly, monthly, or custom interval recurring events
  • Set an end date or number of occurrences for the recurring series
  • Each occurrence can have its own ticket availability
  • Use the [mep_recurring_event] shortcode to display recurring series
  • Attendees can register for individual dates or the full series

How do I show events on a page using shortcodes?

  • Use [mep_event_list] for a list, [mep_event_grid] for a grid
  • Use [mep_calendar] for an interactive calendar
  • Use [mep_event_carousel] for a sliding carousel
  • Add parameters like limit="6", column="3", category="workshop"
  • Place shortcodes in any page, post, or widget area

Setup & Configuration

Answers to setup-related questions about permalinks, licensing, and initial configuration.

Why does my event page show a 404 error?

  • Your permalinks are likely set to "Plain" — go to Settings → Permalinks and choose "Post name"
  • After changing permalinks, click Save to flush rewrite rules
  • If still broken, deactivate and reactivate WpEvently to re-register rewrite rules
  • Check that no other plugin is conflicting with the event post type
  • Verify mod_rewrite is enabled on your server

How do I activate my Pro license?

  • After installing the Pro version, go to WpEvently → License
  • Enter your purchase code (found in your Magepeople account)
  • Click Activate License — a success message confirms activation
  • Licenses are valid for the subscribed domain
  • Deactivate the license before moving sites

Can I use the plugin without WooCommerce?

  • WooCommerce must be installed and active for WpEvently to function
  • The plugin uses WooCommerce for cart, checkout, and payment processing
  • Free events also go through the WooCommerce checkout flow
  • WooCommerce provides the customer and order management backend

How do I change the event slug in URLs?

  • Go to WpEvently → Settings → General Settings
  • Find the Event Slug field and enter your preferred slug
  • Save changes and flush permalinks (Settings → Permalinks → Save)
  • Existing event URLs will change — set up redirects if needed
  • Common slugs: events, sessions, workshops, courses

Attendee & Ticket Management

Frequently asked questions about managing attendees, tickets, and registrations.

Why is the attendee list showing empty?

  • Check that the attendee form is enabled in WpEvently settings
  • Verify that attendees have completed the checkout process
  • Test with a new registration to confirm data is being captured
  • Check for plugin conflicts that may block attendee data storage
  • Ensure the attendee database tables were created during installation

How do I export attendee data to CSV?

  • Go to WpEvently → Attendees in the admin menu
  • Apply any desired filters (event, date range, ticket type)
  • Click the Export CSV button above the table
  • The file downloads with all visible columns plus custom fields
  • Open the CSV in any spreadsheet application

Why don't I see the attendee form on the event page?

  • The attendee form must be enabled in WpEvently → Settings
  • Check that the event has ticket types associated with it
  • Verify the form is not hidden by a custom template override
  • Test with a different theme to rule out theme conflicts
  • Clear any caching plugins that may be serving stale HTML

Can I book a ticket for someone from the backend?

  • Yes — go to WpEvently → Add Booking in the admin menu
  • Select the event, ticket type, and quantity
  • Enter the customer's name, email, and phone number
  • Complete the booking — the customer receives confirmation
  • This is useful for phone orders and walk-in registrations

Troubleshooting

Solutions to common issues encountered while using WpEvently.

How do I enable QR codes on tickets?

  • Install and activate the QR Code addon for WpEvently
  • Go to WpEvently → Settings → QR Code Settings
  • Enable QR code generation and configure size and position
  • QR codes appear on new PDF tickets automatically
  • Existing tickets are not retroactively updated with QR codes

How do I enable the waitlist for sold-out events?

  • Install and activate the Waitlist addon for WpEvently
  • Configure waitlist settings in WpEvently → Settings → Waitlist
  • Enable waitlist per event in the event edit screen
  • When tickets sell out, a "Join Waitlist" button appears
  • Users are notified when tickets become available again

The plugin is not working after an update — what should I do?

  • Check the Magepeople changelog for breaking changes
  • Clear all caching plugins and server cache
  • Flush permalinks by re-saving Settings → Permalinks
  • Deactivate and reactivate WpEvently and all addons
  • If issues persist, restore the previous version from backup

How do I customise the look of my event pages?

  • Use Elementor to design custom single event page layouts
  • Override plugin templates by copying them to your theme
  • Add custom CSS in the WordPress Customizer or theme stylesheet
  • Use the action hooks to inject custom HTML into event templates
  • Install the Pro version for additional styling options

Addon Questions

Common questions about WpEvently addons, their installation, and compatibility.

How do I install addons?

  • Addons are installed like plugins — via zip upload in WordPress
  • Go to Plugins → Add New → Upload Plugin
  • Select the addon .zip file from your Magepeople account
  • Activate the addon and enter your license key if required
  • The base WpEvently plugin must be active for addons to work

Are addons compatible with the free version?

  • Most addons work with both the free and Pro versions of WpEvently
  • Some advanced addons (Seat Plan, Marketplace) may require Pro
  • Check the individual addon documentation for compatibility details
  • Addon license keys are tied to your Magepeople account
  • Test addons on a staging site before using on production

Can I use multiple addons together?

  • Yes — all addons are designed to work together seamlessly
  • For example, use QR Code, Waitlist, and Coupon addons simultaneously
  • Each addon adds its own settings section in the admin panel
  • There are no known conflicts between any WpEvently addons
  • Keep all addons updated to ensure cross-compatibility

How do I get support for addons?

  • Contact Magepeople support through your account dashboard
  • Include your license key and site URL in support requests
  • Check the addon documentation for troubleshooting guides
  • Visit the Magepeople knowledge base for known issues
  • Pro users receive priority support with faster response times
