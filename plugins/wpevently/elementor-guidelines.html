Elementor Widgets Overview

WpEvently registers dedicated Elementor widgets that allow you to place event content anywhere in your Elementor-built pages without touching shortcodes. These widgets are found in the Elementor panel under the WpEvently category.

Available Widgets

  • Event List — Displays events in a vertical list format
  • Event Grid — Responsive card grid with sort and filter options
  • Event Calendar — Interactive month/week/day calendar widget
  • Event Carousel — Horizontal slider with navigation and autoplay
  • Event Minimal — Clean text-only event list for compact spaces
  • Event Native — Native WordPress loop style events

Widget Location

  • Open any page or template in Elementor editor
  • Click the + icon to open the widget panel
  • Search for "WpEvently" or scroll to the WpEvently section
  • Drag the desired widget onto the canvas
  • Configure options in the left sidebar content panel

Using Shortcodes in Elementor

You can also use any WpEvently shortcode inside Elementor pages by placing it in the Shortcode widget or the Text widget. This is useful when you need parameter-level control not exposed in the widget UI.

Shortcode Widget Method

  • Add the "Shortcode" widget from the Elementor panel
  • Paste your WpEvently shortcode into the input field
  • Example: [mep_event_grid column="3" limit="6"]
  • Click Apply — the events render directly in the editor
  • Style the wrapper using Elementor spacing and background options

Text Widget Method

  • Add the "Text" widget to your Elementor canvas
  • Switch to the Text tab in the widget settings
  • Type or paste the shortcode directly into the text editor
  • Example: [mep_calendar category="conference"]
  • Elementor processes the shortcode and displays the output

Single Event Page Design

Design custom single event detail pages using Elementor. You can create unique layouts for each event or a global template that applies to all events.

Building with Elementor

  • Create a new page or edit an existing event post
  • Click "Edit with Elementor" to launch the builder
  • Add WpEvently widgets to display event details, maps, tickets
  • Use dynamic tags to pull event meta fields into text widgets
  • Style the page with Elementor sections, columns, and spacing

Recommended Layout Structure

  • Hero section with event featured image and title overlay
  • Details section with date, time, location, and description
  • Ticket selection area using the event ticket widget
  • Google Map section for venue location
  • Related events section at the bottom using Event Grid widget

Theme Builder Integration

Elementor Pro's Theme Builder allows you to create global templates for single event pages, event archives, and search results. This provides site-wide consistency without per-page editing.

Setting Up Theme Builder

  • Go to Templates → Theme Builder in the Elementor menu
  • Click "Add New" and select "Single" template type
  • Set the preview condition to "WpEvently Events"
  • Design the template using WpEvently widgets and dynamic data
  • Publish the template — all event pages now use this design

Condition & Display Rules

  • Apply to all events: set condition to "All Events"
  • Apply to specific categories: choose "Event Category" condition
  • Create different templates for different event types
  • Use Elementor Pro's dynamic tags for event meta fields
  • Override individual event pages with "Single Event" conditions

Elementor Styling Tips

Optimise your event pages with these styling recommendations. Consistent typography, spacing, and colour choices improve the attendee experience.

Design Best Practices

  • Use a consistent heading hierarchy across all event pages
  • Match event card colours to your brand palette
  • Ensure ticket buttons have sufficient contrast for accessibility
  • Test responsive layouts on mobile and tablet viewports
  • Add hover effects to event cards for visual feedback

Performance Considerations

  • Optimise event featured images before uploading
  • Limit the number of events displayed per page
  • Use lazy loading for event carousel images
  • Minimise custom CSS to reduce render-blocking resources
  • Cache Elementor templates for faster page loads
