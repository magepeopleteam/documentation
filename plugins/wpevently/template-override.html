How to Override Event Templates

WpEvently follows the WordPress template hierarchy. You can override any plugin template by copying it into your theme directory. This preserves your customisations across plugin updates.

Template Override Steps

  • Browse to /wp-content/plugins/wp-event-manager/templates/
  • Locate the template file you want to customise
  • Create a folder in your theme: /wp-content/themes/your-theme/wp-event-manager/
  • Copy the template file, preserving the subdirectory structure
  • Edit the copied file — your changes persist after plugin updates

Template Hierarchy

  • single-event.php — Single event detail page
  • archive-event.php — Event archive listing page
  • content-single-event.php — Content inside single event
  • content-event.php — Content inside event listings
  • ticket/ — All ticket-related template partials

Creating New Theme Templates

You can create entirely new template files for custom event layouts. Use WordPress template hierarchy conventions to target specific event views.

Custom Template Files

  • single-event-{slug}.php — Template for a specific event slug
  • single-event-{id}.php — Template for a specific event ID
  • taxonomy-event_category-{slug}.php — Category-specific archive
  • taxonomy-event_organizer-{slug}.php — Organizer-specific archive
  • Use template-loader.php hooks for advanced custom routing

Template Tags & Functions

  • mep_get_event_title() — Retrieve the event title
  • mep_get_event_date() — Display the event date
  • mep_get_event_time() — Display the event time
  • mep_get_event_location() — Get the venue location
  • mep_get_ticket_price() — Retrieve ticket price info

PDF Ticket Design

Customise the PDF ticket layout that is generated when attendees purchase tickets. You can modify the header, body content, footer, and QR code placement.

PDF Template Override

  • Copy templates/pdf-ticket.php to your theme override folder
  • Edit the HTML structure to match your brand
  • Add your company logo, colours, and typography
  • Modify ticket data fields shown on the PDF
  • Test the PDF output by purchasing a test ticket

PDF Styling Tips

  • Use inline CSS for PDF compatibility
  • Keep the design clean — PDFs are often printed on small paper
  • Include essential info: event name, date, time, venue, attendee name
  • Place the QR code in a prominent, scannable position
  • Add a unique ticket ID and barcode for entry scanning

Social Share Icons

Override the default social share icons on the event details page. You can add, remove, or rearrange share buttons for Facebook, Twitter, LinkedIn, and more.

Customising Share Icons

  • Override templates/social-share.php in your theme
  • Add new networks: Pinterest, WhatsApp, Telegram, Email
  • Use custom icon sets (FontAwesome, SVG sprites, or images)
  • Modify the share URL structure for each platform
  • Add tracking parameters to share URLs for analytics

Available Hook Points

  • mep_before_social_share — Content before share icons
  • mep_after_social_share — Content after share icons
  • mep_social_share_list — Filter the share networks array
  • Use these hooks to inject custom share services
  • Test shared links on each platform for correct URL formatting

Category & Organizer Templates

Override the archive templates for event categories and organizers. These templates control how lists of events grouped by category or organizer are displayed.

Category Archives

  • Copy templates/taxonomy-event_category.php to your theme
  • Customise the header to show the category name and description
  • Change the layout style (list, grid, or custom)
  • Add category-specific banners or branding
  • Include subcategory navigation for hierarchical categories

Organizer Archives

  • Copy templates/taxonomy-event_organizer.php to your theme
  • Show organizer profile details at the top of the page
  • Display all events by that organizer in a grid or list
  • Add links to organizer website and social profiles
  • Include an organizer bio or description section

Template Override Best Practices

Follow these guidelines to ensure your template overrides remain maintainable, compatible, and performant.

Maintenance Tips

  • Document each overridden template in a readme file
  • Review overridden templates after each plugin update
  • Use a child theme to keep customisations separate
  • Version control your theme files with Git
  • Test on a staging site before deploying to production

Compatibility Checklist

  • Verify that all required template tags still exist after updates
  • Check for new template files added by plugin updates
  • Ensure your overridden template uses the same PHP version features
  • Test with both free and Pro versions of the plugin
  • Validate HTML output with W3C validator
