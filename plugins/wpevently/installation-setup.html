Installation Methods

Install the WpEvently plugin using one of three standard WordPress methods. Each approach achieves the same result, so pick the one that suits your workflow.

Via WordPress Admin

  • Navigate to Plugins → Add New in your WordPress dashboard
  • Search for "WpEvently" or "WooCommerce Event Manager"
  • Click Install Now, then Activate once the installation finishes
  • The plugin will appear in your installed plugins list
  • Pro version requires manual upload — see zip method below

Via Zip Upload

  • Download the plugin .zip file from your Magepeople account
  • Go to Plugins → Add New → Upload Plugin
  • Choose the downloaded .zip file and click Install Now
  • Click Activate Plugin after the upload completes
  • This method is required for the Pro version installation

Via FTP

  • Extract the plugin .zip file on your local machine
  • Connect to your server using an FTP client (FileZilla, WinSCP, etc.)
  • Upload the extracted folder to /wp-content/plugins/
  • Return to WordPress admin and go to Plugins
  • Find WpEvently in the list and click Activate

Pro Version Activation

  • Purchase the Pro version from the Magepeople website
  • Download the pro plugin .zip from your account dashboard
  • Install via zip upload — do not install both free and pro
  • Go to WpEvently → License in the admin menu
  • Enter your purchase code and click Activate License

System Requirements

Ensure your server meets the following minimum requirements before installing the plugin. Meeting these guarantees smooth operation and compatibility.

WordPress & PHP

  • WordPress 5.8 or higher (6.x recommended)
  • PHP version 7.4 or later (PHP 8.0+ recommended)
  • MySQL 5.6+ or MariaDB 10.1+
  • WooCommerce plugin installed and active (4.0+)
  • Memory limit of 128 MB or more (256 MB recommended)

Server Extensions

  • mod_rewrite enabled (for clean permalinks)
  • GD Library or Imagick for image processing
  • cURL extension enabled for API communications
  • JSON extension for REST API functionality
  • MBString extension for multi-byte character support

Permalink Configuration

WpEvently requires clean permalinks for event pages, ticket URLs, and calendar views. The plugin will not function correctly with the default "Plain" permalink structure.

Setting Up Permalinks

  • Go to Settings → Permalinks in your WordPress dashboard
  • Select the "Post name" option for best compatibility
  • Click Save Changes to flush the rewrite rules
  • Visit an event page to confirm URLs resolve correctly
  • If you see a 404, navigate to Settings → Permalinks again and re-save

Custom Structure Options

  • /%postname%/ — Recommended default structure
  • /events/%postname%/ — Events under a dedicated prefix
  • /%category%/%postname%/ — Includes event category in URL
  • Always flush permalinks after changing the structure
  • Test a few event and ticket pages after each change

Post-Installation Steps

After installing and activating the plugin, follow these steps to configure your first events and ensure everything is working correctly.

Initial Wizard

  • Run the guided setup wizard from the WpEvently dashboard
  • Configure global currency, date format, and timezone
  • Set default ticket settings and registration options
  • Choose email notification preferences
  • Save settings and verify the configuration summary

Verification Checklist

  • Create a test event and publish it
  • View the event on the front end to verify the URL works
  • Test the ticket purchase flow with a sandbox payment
  • Check that attendee data appears in the admin panel
  • Confirm email notifications are sent for test registrations
