Events List Style Shortcode

Display events in a vertical list layout with event thumbnails, dates, titles, and ticket buttons. Ideal for sidebars, agendas, and schedule pages.

Basic Usage

  • [mep_event_list] — Shows all upcoming events in list format
  • [mep_event_list limit="6"] — Limits to 6 events
  • [mep_event_list show="3"] — Shows 3 events per page with pagination
  • [mep_event_list category="conference"] — Filters by event category slug
  • [mep_event_list order="asc"] — Sorts events in ascending date order

Parameters

  • limit — Number of events to display (default: all)
  • show — Events per page for paginated lists
  • category — Filter by event category slug
  • order — Sort order: asc or desc
  • style — Layout style: list or grid

Events Grid Style Shortcode

Display events in a responsive card grid. Each card shows the event thumbnail, date, title, location, and a book now button.

Basic Usage

  • [mep_event_grid] — Shows all upcoming events in grid layout
  • [mep_event_grid column="3"] — Sets grid to 3 columns
  • [mep_event_grid column="4" limit="8"] — 4 columns showing 8 events
  • [mep_event_grid category="workshop"] — Filters by category slug
  • [mep_event_grid show_footer="no"] — Hides the footer area

Parameters

  • column — Number of grid columns (2, 3, or 4, default: 3)
  • limit — Number of events to display
  • category — Filter by event category slug
  • show_footer — Show/hide the card footer (yes/no)
  • order — Sort order: asc or desc

Common Shortcode Parameters

These parameters are shared across multiple shortcodes and control global display behaviour such as date formats, filters, and styling.

Date & Filtering

  • from_date — Show events starting from this date (YYYY-MM-DD)
  • to_date — Show events up to this date (YYYY-MM-DD)
  • show_past — Include past events in results
  • hide_expired — Automatically hide expired events
  • city — Filter events by city name

Display Controls

  • column — Number of grid columns (2-4)
  • order — Sort direction: asc or desc
  • show_footer — Show/hide card footer elements
  • pagination — Enable or disable pagination
  • class — Additional CSS class for custom styling

How to Define Column Numbers in Grid-Style Event List

The column parameter controls how many columns appear in the grid-style layout.

Basic Usage

  • [event-list style='grid' column='3'] — 3-column grid
  • [event-list style='grid' column='4'] — 4-column grid
  • [event-list style='grid' column='3' limit='9'] — 3 columns with 9 events
  • Supports 3 or 4 columns

Notes

  • Default value is 3 if not specified
  • Only applies when style='grid' is used
  • Responsive: columns stack on smaller screens
