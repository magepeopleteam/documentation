Attendee List

The attendee list shows all people who have registered for an event. You can view, search, filter, and manage attendees from the WordPress admin panel.

Viewing Attendees

  • Go to WpEvently → Attendees in the admin menu
  • All attendees across all events are listed in a table
  • Filter by event, date range, ticket type, or status
  • Click an attendee name to view their full details
  • Use the search box to find specific attendees by name or email

Attendee Details

  • Full name, email address, and phone number
  • Ticket type purchased and quantity
  • Order ID and purchase date
  • Check-in status for event day management
  • Custom attendee form fields if enabled

Export Attendee List to CSV

Export your attendee data to a CSV file for use in spreadsheets, email marketing tools, or external systems. The export includes all visible columns plus custom fields.

How to Export

  • Navigate to WpEvently → Attendees
  • Apply any filters you need (specific event, date range, etc.)
  • Click the Export CSV button above the attendee table
  • The CSV file downloads automatically to your computer
  • Open the file in Excel, Google Sheets, or any CSV reader

CSV File Structure

  • Columns include: Name, Email, Phone, Event Name, Ticket Type, Quantity, Order ID, Purchase Date, Status
  • Custom attendee form fields are appended as additional columns
  • Dates are formatted in YYYY-MM-DD HH:MM:SS format
  • UTF-8 encoding ensures special characters display correctly
  • Filtered exports only include attendees matching the current filters

Backend Booking (Book for Customer)

Site administrators can book tickets on behalf of customers directly from the WordPress admin. This is useful for phone orders, walk-in registrations, or manual adjustments.

Booking from Admin

  • Go to WpEvently → Add Booking from the admin menu
  • Select the event you want to book tickets for
  • Choose the ticket type and quantity
  • Enter the customer's name, email, and phone number
  • Complete the booking — the customer receives confirmation

Use Cases

  • Phone orders from customers who prefer to call
  • Walk-in registrations at the event venue
  • Comping tickets or complimentary registrations
  • Transferring bookings between ticket types
  • Testing the booking flow for quality assurance

Attendee Registration Form

The attendee form collects additional information from registrants during checkout. You can enable, disable, and customise which fields appear on the form.

Enabling the Form

  • Go to WpEvently → Settings → General Settings
  • Find the Attendee Form section
  • Toggle the option to Enable Attendee Form
  • Choose which fields are required or optional
  • Save changes and test the checkout flow

Custom Fields

  • Add custom fields like company name, job title, dietary requirements
  • Field types: text, email, phone, number, dropdown, textarea, checkbox
  • Reorder fields by dragging them in the settings panel
  • Set conditional logic to show/hide fields based on ticket type
  • Exported CSV includes all custom attendee form fields

Event Page Shows 404 Error

If your event pages return a 404 error despite correct settings, the issue is almost always related to permalink configuration. Follow these steps to resolve it.

Common Causes & Fixes

  • Permalinks set to "Plain" — change to "Post name" or custom structure
  • Rewrite rules need flushing — go to Settings → Permalinks and click Save
  • Event slug conflicts with existing page or post slugs
  • Plugin activated before WooCommerce — deactivate and reactivate WpEvently
  • Custom post type rewrite rules not registered — flush permalinks again

Advanced Troubleshooting

  • Check .htaccess file exists and is writable
  • Verify mod_rewrite is enabled on the Apache server
  • Check for plugin conflicts by deactivating other plugins
  • Switch to a default WordPress theme to rule out theme issues
  • Use the WordPress health check tool to inspect rewrite rules

QR Codes on Tickets

QR codes on tickets enable quick attendee check-in at the event venue. Each ticket generates a unique QR code that can be scanned using a smartphone or dedicated scanner.

Enabling QR Codes

  • Install and activate the QR Code addon for WpEvently
  • Go to WpEvently → Settings → QR Code Settings
  • Enable QR code generation on PDF tickets
  • Choose QR code size, position, and error correction level
  • Save settings — new tickets will include QR codes automatically

Scanning & Check-in

  • Use any QR code scanner app on iOS or Android
  • Scan the QR code on the attendee's ticket (printed or digital)
  • The scan validates the ticket and marks the attendee as checked in
  • Check-in status is updated in the attendee list in real time
  • Prevent duplicate entry with one-time scan validation

Waitlist for Sold-Out Events

When an event reaches full capacity, the waitlist feature allows interested attendees to register for a spot. They are automatically notified when tickets become available.

Enabling Waitlist

  • Install and activate the Waitlist addon
  • Go to WpEvently → Settings → Waitlist Settings
  • Enable the waitlist feature globally or per event
  • Configure notification settings for waitlist members
  • Set maximum waitlist size if needed

Waitlist Management

  • Attendees join the waitlist from the event page when tickets are sold out
  • Email notifications are sent when new tickets become available
  • Waitlist positions are allocated on a first-come, first-served basis
  • Admin can view and manage the waitlist from the event edit screen
  • Expired waitlist entries are automatically cleaned up
