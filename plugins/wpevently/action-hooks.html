Social Share Button Hooks

WpEvently provides hooks around the social share buttons section, allowing you to inject additional share networks, custom content, or analytics tracking.

Available Hooks

  • mep_before_social_share — Fires before the social share buttons
  • mep_after_social_share — Fires after the social share buttons
  • mep_social_share_list — Filter hook to modify the share networks array
  • mep_social_share_item — Action hook for each individual share button
  • Use these to add custom share services or promotional banners

Usage Example

  • add_action('mep_before_social_share', 'my_custom_share_header');
  • Define a function that echoes HTML before the share buttons
  • Add WhatsApp, Telegram, or email share options via the filter
  • Wrap tracking pixels or analytics code in hook callbacks
  • Test that injected content renders correctly on event pages

Google Map Section Hooks

Customise the Google Map section on event detail pages. Hooks are available to add content before and after the map, or to modify map settings.

Available Hooks

  • mep_before_google_map — Inject content above the map
  • mep_after_google_map — Inject content below the map
  • mep_google_map_args — Filter the map configuration array
  • mep_google_map_marker — Filter marker position and info window
  • Modify zoom level, map type, and styling via mep_google_map_args

Map Customisation

  • Change map height and width via CSS on the map container
  • Add multiple markers for events with multiple locations
  • Customise the info window content with event details
  • Set custom map styles (snazzymaps JSON for branded look)
  • Disable the map entirely by unhooking the default callback

Adding Custom Functions

Extend WpEvently by adding custom PHP functions to your theme's functions.php file or a site-specific plugin. Use the available action and filter hooks to modify behaviour.

Common Customisations

  • Modify event query parameters using pre_get_posts and mep_event_query_args
  • Add custom fields to the event creation form
  • Redirect users after ticket purchase using mep_ticket_purchase_redirect
  • Send custom email notifications on event registration
  • Validate attendee data before processing the booking

Example Snippet

  • add_filter('mep_event_query_args', function($args) { $args['posts_per_page'] = 10; return $args; });
  • Always use a prefix for your function names to avoid conflicts
  • Check for function existence before declaring custom functions
  • Use is_plugin_active() to conditionally load hook code
  • Test customisations after every plugin update

Hook into Ticket Rendering

The ticket selection and rendering process exposes hooks that let you modify ticket display, pricing, availability, and checkout behaviour.

Ticket Display Hooks

  • mep_before_ticket_type — Content before the ticket type selector
  • mep_after_ticket_type — Content after the ticket type selector
  • mep_ticket_type_item — Modify each ticket type row output
  • mep_ticket_price_html — Filter the displayed price HTML
  • Add quantity limits, custom descriptions, or promotional badges

Booking & Cart Hooks

  • mep_before_add_to_cart — Before adding ticket to cart
  • mep_after_add_to_cart — After ticket is added to cart
  • mep_ticket_validation — Custom validation rules for ticket selection
  • mep_cart_item_data — Filter cart item data for tickets
  • Integrate with loyalty programmes or membership discounts

Complete Hook Reference

A comprehensive list of all action and filter hooks available in WpEvently. Use this as a quick reference when building custom integrations.

Action Hooks

  • mep_before_event_list — Before event list output
  • mep_after_event_list — After event list output
  • mep_before_single_event — Before single event content
  • mep_after_single_event — After single event content
  • mep_before_ticket_form — Before the ticket booking form
  • mep_after_ticket_form — After the ticket booking form

Filter Hooks

  • mep_event_list_args — Modify event list query arguments
  • mep_single_event_template — Override single event template path
  • mep_ticket_price — Modify ticket price before display
  • mep_attendee_data — Filter attendee information on submission
  • mep_email_subject — Customise email notification subjects
